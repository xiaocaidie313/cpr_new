# 05 · 急救 UI

位于 `ui/`。基于 Jetpack Compose，遵循**无状态组件 + 状态外置**原则。

---

## 设计原则（急救场景特化）

施救者处于高压、可能强光/暗光、手忙脚乱，所以：

- **大字**：核心指导字号 44sp、加粗、动词开头（如「用力按压 / 胸部中央」）。
- **高对比**：深色背景 + 纯白文字（`EmergencyPalette`），独立于系统主题，任何环境都可读。
- **强语义色**：红=危险、黄=警告、绿=达标，无需阅读即可判断。
- **少决策**：主屏只有「开始 / 拨打 120 / 结束」三个大按钮。

---

## 组件（`ui/component/CprComponents.kt`，全部无状态）

| 组件 | 作用 |
| --- | --- |
| `StatusBanner` | 顶部阶段标签 + 兜底告警横幅（可点击关闭） |
| `GuidanceCard` | 大字指导卡片（急救主视图核心） |
| `MetronomeIndicator` | 节拍脉冲圆，按 bpm 做缩放呼吸动画 |
| `QualityDashboard` | 质量分 / 频率 / 手位三块指标，按状态变色 |

> 节拍脉冲用 `rememberInfiniteTransition` 按 bpm 近似，而非逐拍重绘，性能友好。

## 主屏（`ui/screen/CprGuidanceScreen.kt`）

- 无状态：所有数据来自 `CprSessionState`，所有交互通过回调上抛。
- 两态：
  - **待命态** `IdleContent`：大号「开始急救」按钮 + 说明。
  - **进行态** `ActiveContent`：横幅 + 大字指导 + 仪表盘 + 节拍器 + 底部「拨打 120 / 结束」。
- 会话结束弹出 `HandoverDialog` 交接报告。

## 装配（`MainActivity.kt`）

```kotlin
val deps = remember { ServiceLocator.provideDependencies(context) }
val vm: CprSessionViewModel = viewModel(factory = CprSessionViewModel.factory(deps))
val state by vm.state.collectAsStateWithLifecycle()
CprGuidanceScreen(state, onStart = { ... vm.startSession() }, ...)
```

开始急救时顺带触发权限请求；即使被拒，会话也会降级运行。

---

## 可扩展方向

- 多页导航：当前单屏，后续可换 `NavHost`，屏幕仍保持无状态即可平滑迁移。
- 摄像头预览：接入 CameraX 后可在主视图叠加 `PreviewView`（见硬件文档）。
- 主题：高对比配色集中在 `EmergencyPalette`，改色只动一处。
